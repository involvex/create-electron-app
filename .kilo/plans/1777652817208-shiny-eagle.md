# Plan: Fix create-electron-app CLI - Implement Missing Scaffolding Logic

## Problem Diagnosis

**Root Cause**: The CLI is non-functional because `src/index.ts` contains only basic path setup (10 lines). Despite commit `cf52c88` claiming "add interactive Bun Electron scaffolding tool", the actual implementation is missing. No prompts, no templates, no file generation exists.

**Evidence**:
- `src/index.ts`: 10 lines (just imports and `_PROJECT_ROOT` constant)
- No `templates/` directory exists
- No file generation logic
- README documents many features but code is absent

## Solution Approach

Build the complete interactive CLI from scratch based on the documented requirements in README.md and README_CLI.md, using @clack/prompts for UI and embedding templates directly in the codebase.

## Implementation Steps

### Phase 1: Core CLI Infrastructure

1. **Install @clack/prompts** (already in dependencies)
2. **Create `src/cli/prompts.ts`** - Define all interactive prompts:
   - `promptProjectName()` - Text input with validation (no spaces, valid npm package name)
   - `promptTemplate()` - Select: vanilla, react, react-tailwind, vue
   - `promptStyling()` - Select styling framework (Tailwind, Aura UI, Kumo, shadcn/ui, custom CSS)
   - `promptUIFeatures()` - Multi-select: glass morphism, navbar type, tray menu, theme
   - `promptCodeQuality()` - Multi-select: ESLint, Prettier, Git init, Husky hooks
   - `promptBackend()` - Select: Vite backend, Bun.serve, Express, None
3. **Create `src/cli/constants.ts`** - Define all option constants:
   - Template types enum
   - Styling frameworks list
   - Navigation options
   - Theme presets
4. **Create `src/cli/validation.ts`** - Validation functions:
   - Project name (no spaces, valid identifier, not reserved)
   - Directory existence check
   - Overwrite confirmation

### Phase 2: Template System

1. **Create `src/templates/` directory** with subdirectories for each template type
2. **Create template files** for each project structure:
   - `package.json.tmpl` - with all dependencies based on choices
   - `tsconfig.json.tmpl` - strict config
   - `vite.main.config.ts.tmpl` - Vite config for main process
   - `vite.renderer.config.ts.tmpl` - Vite config for renderer
   - `electron.vite.config.ts.tmpl` - electron-vite combined config (if using electron-vite)
   - `bunfig.toml.tmpl` - Bun configuration
   - `.gitignore.tmpl`
   - `src/main/index.ts.tmpl` - Electron main process entry
   - `src/main/menu.ts.tmpl` - Menu generation (native or custom)
   - `src/preload/index.ts.tmpl` - Preload script with context bridge
   - `src/renderer/index.html.tmpl` - HTML entry (with theme toggle if selected)
   - `src/renderer/main.tsx` or `main.ts` or `main.vue` - Frontend entry
   - `src/renderer/App.tsx` / `App.vue` / `app.ts` - Main app component
   - `src/renderer/index.css.tmpl` - Global styles with glass morphism classes
   - `src/shared/types.ts.tmpl` - Shared TypeScript types
   - `tailwind.config.js.tmpl` (if Tailwind selected)
   - `eslint.config.js.tmpl` (if ESLint selected)
   - `prettier.config.js.tmpl` (if Prettier selected)

3. **Template variables**: Use simple placeholder replacement:
   - `{{projectName}}` - User's project name
   - `{{projectDescription}}` - Default description
   - `{{author}}` - From git config or placeholder
   - `{{template}}` - Template type
   - `{{styling}}` - Selected styling framework
   - `{{hasGlassMorphism}}` - boolean
   - `{{navigationType}}` - navbar option
   - `{{hasTray}}` - boolean
   - `{{theme}}` - theme preset
   - `{{backend}}` - backend selection

### Phase 3: File Generation Engine

1. **Create `src/generators/project.ts`**:
   - `generateProject(options: ProjectOptions)` - Main orchestrator
   - Create target directory
   - Copy template files with placeholder substitution
   - Apply post-processing (chmod +x for scripts)
   - Initialize git repo if selected
   - Print success message with next steps

2. **Create `src/generators/configs.ts`**:
   - `generatePackageJson(options)` - Dynamic package.json based on template + features
   - `generateViteConfigs(options)` - Vite configs for main/renderer
   - `generateTsConfig(options)` - TypeScript config
   - `generateBunfig(options)` - Bun configuration

3. **Create `src/generators/menu.ts`**:
   - `GenerateMenuCode(options)` - Generates menu.ts with native menu or custom navbar HTML
   - Tray menu code if enabled
   - Theme toggle integration

4. **Create `src/generators/themes.ts`:
   - `GenerateThemeCSS(options)` - CSS variables for selected theme
   - Glass morphism utility classes

5. **Create `src/generators/git.ts`:
   - `InitializeGit(projectPath)` - Run `git init`, `git add .`, `git commit`

### Phase 4: Main CLI Orchestration

**Rewrite `src/index.ts`**:

```typescript
#!/usr/bin/env bun

import { generateProject } from "./generators/project";
import { collectUserInput } from "./cli/prompts";
import { intro, outro } from "@clack/prompts";

async function main() {
  intro({ message: "Create Electron App with Bun" });

  try {
    const options = await collectUserInput();
    await generateProject(options);
    outro({ message: "Project created successfully!" });
  } catch (error) {
    console.error("Error:", error);
    process.exit(1);
  }
}

main();
```

## Technical Details

### Placeholder Substitution

Simple approach: Read template file as string, replace `{{variable}}` with actual values using regex. For JSON files (package.json), parse as JSON, interpolate values, then stringify.

### Electron-Vite vs Vite + Plugin

Based on plan: Use **electron-vite** for simplicity. Generate:
- `electron.vite.config.ts` as main config
- Or separate `vite.main.config.ts` + `vite.renderer.config.ts` if using Vite directly

Recommendation: Use electron-vite for better HMR and simpler config.

### Glass Morphism CSS

Provide utility classes:

```css
.glass {
  background: rgba(255, 255, 255, var(--glass-opacity, 0.1));
  backdrop-filter: blur(var(--glass-blur, 16px));
  -webkit-backdrop-filter: blur(var(--glass-blur, 16px));
  border: 1px solid rgba(255, 255, 255, var(--glass-border, 0.2));
  border-radius: var(--glass-radius, 16px);
}
.glass-soft { --glass-blur: 8px; --glass-opacity: 0.1; }
.glass-standard { --glass-blur: 16px; --glass-opacity: 0.15; }
.glass-strong { --glass-blur: 24px; --glass-opacity: 0.2; }
```

### Theme System

CSS variables for colors:

```css
:root {
  --bg-primary: #ffffff;
  --text-primary: #1a1a1a;
  --glass-bg: rgba(255,255,255,0.1);
}
[data-theme="dark"] {
  --bg-primary: #0f172a;
  --text-primary: #f1f5f9;
  --glass-bg: rgba(0,0,0,0.3);
}
```
