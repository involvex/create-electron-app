# Plan: Modern CLI for create-electron-app with Bun Runtime

## Overview

Build a modern interactive CLI tool (`bunx @involvex/create-electron-app`) that scaffolds Electron applications using Bun runtime, TypeScript, with full-stack bundling, hot reloading, and extensive customization options.

## Research Findings - Modern Stack

### Bun + Electron Integration

- **Bun 1.3+** provides `--watch` (hard restart) and `--hot` (soft reload) modes for development
- `Bun.build()` supports bundling with native file watching capabilities
- Bun's built-in bundler is 10-100x faster than webpack/esbuild for many workloads
- Hot Module Replacement (HMR) works via `import.meta.hot` API (Vite-compatible)
- Bun can compile to standalone executables via `bun build --compile`

### Bundling Options

1. **Vite + Bun** - Fastest HMR, mature ecosystem, `vite-plugin-electron` support
   - `vite-plugin-electron` handles dual environment (main + renderer)
   - Supports multi-process HMR out of the box
   - Electron Forge has official Vite plugin
2. **Bun Native Bundler** - Zero-dependency, ultra-fast, but less mature for Electron
3. **Webpack/ESBuild** - Traditional but slower

**Recommendation**: Use Vite with Bun runtime for best developer experience and HMR performance

### CSS/Styling Frameworks

Modern CSS-in-JS and utility-first options:

**Primary Recommendations:**

- **Tailwind CSS 4** with `@theme` tokens - Industry standard, glass morphism support
- **Aura UI** - Built on Tailwind 4, "Vibrant Depth" design language, glass morphism utilities
- **shadcn/ui** - Modern React component library (50+ components) with Tailwind, full customization
- **@cloudflare/kumo** - Cloudflare's design system (Base UI + Tailwind v4), 87K weekly downloads, 25+ components
- **@casoon/tailwindcss-glass** - Complete glass morphism system (69KB minified, all CSS)
- **sleyt** - Minimal pure CSS framework with glass morphism, dark mode

**Lightweight Alternatives (no-build, pure CSS, <35KB):**

- **µCSS** - ~19KB gzipped, CSS-only, semantic HTML, 20+ color themes
- **Symphony CSS** - ~35KB, semantic-first, zero-class baseline, OKLCH colors
- **Graffiti** - ~26KB minimal, drop-in toolkit, fluid typography
- **Lissom.CSS** - Classless, minimalist, themeable
- **Frankenstyle** - No-build, utility-first, single stylesheet
- **Cutestrap** - 2.7KB, progressive enhancement
- **SamphireCSS** - 12KB gzipped, content-first

**Web Design 2026**: Glassmorphism + dark mode + gradients + micro-animations

### Menu Options

- **Native Electron Menu** - `Menu.buildFromTemplate()` for app menu, tray, context menus
- **Custom HTML/CSS menubar** - Full customization, requires IPC for native operations
- **Tray menus** - System notification area, cross-platform
- **maxogden/menubar** library - Wrapper for tray-based menubar apps

**Recommendation**: Offer both native Electron Menu AND custom top navbar option with sticky positioning

### Interactive CLI Tools

- **@clack/prompts** - Beautiful, minimal UI (80% smaller than alternatives), TypeScript-first
  - Components: text, password, confirm, select, multiselect, autocomplete, spinner, tasks
  - Built-in intro/outro, cancellation handling
- **@tsed/cli-prompts** - Wraps @clack/prompts with dependency injection

### Hot Reloading Setup

**electron-vite approach (recommended):**

- Instant HMR for renderer process
- Hot reloading for main process
- Automatic process management

**Vite + electron-vite-plugin:**

```ts
import electron from "vite-plugin-electron/simple";

export default {
  plugins: [
    electron({
      entry: "src/main/index.ts",
      onstart({ reload }) {
        // Custom logic on restart
      },
    }),
  ],
};
```

**Bun native watch:**

```json
{
  "dev": "concurrently \"bun --watch src/main/index.ts\" \"bunx --bun vite\""
}
```

  if (process.platform !== "darwin") app.quit();
});

## Technical Decisions

### Vite vs Electron-Vite

**electron-vite**:

- ✅ Purpose-built for Electron
- ✅ Single config point
- ✅ Better HMR for main process
- ✅ Active maintenance
- ❌ Smaller community than Vite alone

**Vite + vite-plugin-electron**:

- ✅ More flexible
- ✅ Larger ecosystem
- ✅ Better docs
- ❌ More complex config

**Decision**: Use **electron-vite** for simplicity and better Electron integration

### Bun vs npm/yarn

**Bun**:

- ✅ 10-100x faster installs
- ✅ Built-in bundler
- ✅ Native TypeScript support
- ✅ Watch/hot modes
- ❌ Less mature ecosystem

**Decision**: Default to **Bun**, fallback to npm support

### Tailwind 4 + Aura UI

- Modern CSS-in-JS approach
- Design tokens via @theme
- Glass morphism built-in
- Vibrant depth design language
- Dark mode first

### Glass Morphism Variants

1. **Soft Glass**: Light blur (8px), subtle (bg-white/10)
2. **Standard Glass**: Medium blur (16px), moderate (bg-white/15)
3. **Strong Glass**: Heavy blur (24px), dense (bg-white/20)
4. **Dark Glass**: Dark mode variants with blue/purple tints
5. **Gradient Glass**: Gradient backgrounds behind glass

### Navigation Patterns

1. **Native Menu**: Electron MenuBar (File, Edit, View, Window, Help)
2. **Sticky Navbar**: Top-0 fixed/sticky with glass effect
3. **Tray Menu**: System notification area
4. **Context Menu**: Right-click menus

## Risk Assessment

**High Risk:**

- Bun + Electron compatibility (mitigated: well-tested combo)
- HMR reliability (mitigated: use electron-vite, proven solution)

**Medium Risk:**

- Glass morphism performance on low-end devices (mitigated: provide fallback)
- Tray icon issues on Linux (mitigated: use standard formats)

**Low Risk:**

- CSS framework updates (mitigated: use stable versions)
- Electron breaking changes (mitigated: pin versions, follow releases)
