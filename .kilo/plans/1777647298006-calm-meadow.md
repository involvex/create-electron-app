# Agent Instructions for create-electron-app

## Development Setup
- Install dependencies: `bun install`
- Run CLI locally: `bun run src/index.ts` (or `bun dev`)
- Global usage: `bunx @involvex/create-electron-app`

## Essential Commands
- Format code: `bun run format`
- Lint code: `bun run lint`
- Type check: `bun run typecheck`
- Full check: `bun run check` (runs format, lint, typecheck)
- Start dev server: `bun run src/index.ts` (interactive CLI)

## Code Quality Workflow
1. Format: `bun run format`
2. Lint: `bun run lint`
3. Type check: `bun run typecheck`
4. Full verification: `bun run check`

## Project Structure
- Main CLI entry: `src/index.ts`
- Configuration files in root: `tsconfig.json`, `biome.json`, `package.json`
- Generated app structure includes:
  - `src/main/` - Electron main process
  - `src/renderer/` - Renderer process (React/Vue/Vaniilla)
  - `src/preload/` - Preload scripts
  - `src/shared/` - Shared TypeScript types
  - Vite configs: `vite.main.config.ts`, `vite.renderer.config.ts`, `electron.vite.config.ts`
  - Styling configs: `tailwind.config.js` (if Tailwind selected)
  - Code quality: `eslint.config.js`, `prettier.config.js` (if selected)

## Framework-Specific Details
- Uses Bun runtime (not Node.js) - all commands use `bun`
- TypeScript strict mode enabled
- Biome for linting/formatting (not ESLint/Prettier directly)
- electron-vite for HMR and building
- Templates: Vanilla, React, Vue, React+Tailwind
- Styling options: Tailwind, Kumo, shadcn/ui, Aura UI, CSS alternatives
- Glass morphism and theme system built-in

## Verification
- No test command defined in package.json - testing is done via generated apps
- Focus on code quality: format -> lint -> typecheck
- Electron apps built with: `electron-vite build`
